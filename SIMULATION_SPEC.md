# 模擬規格

## 戰力與勝率

戰力值越小代表球員越強。單點勝率使用 Logistic 函數：

```text
P(x beats y) = 1 / (1 + exp(s * (x - y)))
```

- `x`：A 方該點戰力。
- `y`：B 方該點戰力。
- `s`：敏感度，預設 `1.0`。

當兩方戰力相同時，勝率為 `0.5`。當 A 方戰力數字較小時，A 方勝率高於 `0.5`。

## 排點

每隊 7 人分配到五點：

1. 單打
2. 雙打
3. 單打
4. 雙打
5. 單打

所有 7 人都必須使用一次，不能重複。

## 雙打

第一版雙打戰力為兩名球員戰力平均值：

```text
doubles strength = (p1 + p2) / 2
```

## 團體賽勝率

每一點先計算 A 方勝率，再計算 A 方在五點中至少拿下三點的機率。

## 策略

- 單點思維：逐點貪婪選擇，優先提高當前點勝率。
- 多點思維：枚舉所有合法排點，選出整體三勝機率最高的排點。

第一版先支援「已知對手排點」情境。雙方同時最佳反應、資訊不完全與學生行為模型留到第二版。

## 完整對戰模擬

目前介面支援兩層分析：

- 整體對戰：所有隊伍彼此對戰，產生勝率矩陣與平均勝率排名。
- 單一對戰：選定兩隊後，手動配置雙方五點排點，並和單點思維、多點思維比較。

整體對戰中的策略比較採「同策略互打」：

- Single Scenario：雙方都用單點思維產生排點。
- Multi-Scenario：雙方都用多點思維產生排點。

這個設定是課堂展示用的策略比較，不宣稱是 Nash equilibrium。

## Monte Carlo 多輪模擬

除了理論勝率，系統也會依五點勝率抽樣模擬多場五戰三勝比賽。

- 預設模擬次數：10,000 次。
- 顯示模擬勝率與比分分布，例如 3-0、3-1、3-2。
- 使用固定 seed 的 pseudo-random generator，讓測試可重現。
